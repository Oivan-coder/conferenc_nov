> Актуализация от 2026-04-09: проект больше не использует `script.js` как точку входа. Общие UI-инициализации выполняются из `js/common.js`, главная страница использует `js/main.js`, а конференции и страница "О нас" переведены на отдельные модули в `js/modules/`.

- Общие интеракции сайта (`burger menu`, плавный скролл, кнопка "наверх") теперь централизованы в `js/common.js`
- Конференц-логика вынесена в `js/modules/conference.js`
- Логика страницы "О нас" живёт в `js/modules/modal.js`
- `script.js` удалён из активной архитектуры

Ниже в документе могут встречаться исторические упоминания старой схемы; их стоит воспринимать как архивный контекст, а не как описание текущего состояния.

## 📋 ОБЩЕЕ ОПИСАНИЕ

Официальный сайт Референс-центра лабораторной службы Московской области - это информационный портал для организации и проведения медицинских конференций, форумов и образовательных мероприятий в сфере лабораторной диагностики.

**Основное назначение:**
- Информирование о предстоящих и прошедших мероприятиях
- Регистрация участников на конференции
- Представление команды и спикеров
- Контактная информация и обратная связь

### Архитектурные решения

**✅ Сильные стороны:**
1. **Модульная CSS архитектура** - разделение на компоненты и секции
2. **Единая shell-оболочка** - header и footer собираются через `js/common.js`
3. **Стабильные html-маршруты** - clean URL rewrite отключён, чтобы не плодить битые адреса
4. **SEO-оптимизация** - мета-теги, Open Graph, sitemap.xml

**⚠️ Проблемы:**
1. **Переходное состояние JS-архитектуры** - часть страниц уже на новом слое, часть ещё на legacy
2. **Большой монолитный файл** - `script.js` всё ещё содержит разнотипную логику нескольких страниц
3. **Устаревшая документация** - часть утверждений ниже относится к прошлой структуре проекта

## ⚠️ ПРОБЛЕМЫ И ОБЛАСТИ ДЛЯ УЛУЧШЕНИЯ

### Критичные (исправлены согласно AUDIT_REPORT.md)
- ✅ XSS уязвимости - **ИСПРАВЛЕНО**
- ✅ Inline обработчики - **ИСПРАВЛЕНО**
- ✅ Console.log в production - **ИСПРАВЛЕНО**

### Средний приоритет

#### 1. Дублирование кода
**Проблема:** Функции `initBurgerMenu()`, `initSmoothScroll()`, `initBackToTop()`, `showNotification()` дублируются в `script.js` и `js/main.js`.

**Решение:**
- Создать общий модуль `js/common.js`
- Импортировать в оба файла
- Или использовать один основной файл

**Оценка:** 1-2 часа

#### 2. Большой монолитный файл
**Проблема:** `script.js` содержит 1200+ строк кода.

**Решение:**
- Разделить на модули:
  - `js/modules/speakers.js` - работа со спикерами
  - `js/modules/map.js` - работа с картами
  - `js/modules/modal.js` - модальные окна
  - `js/modules/forms.js` - формы
- Использовать ES6 modules или bundler

**Оценка:** 6-8 часов

#### 3. Оптимизация загрузки скриптов
**Проблема:**
- Все скрипты загружаются синхронно
- Яндекс.Карты API загружается на всех страницах (нужен только на некоторых)

**Решение:**
- Добавить `defer` или `async` для некритичных скриптов
- Загружать Яндекс.Карты только на страницах с картами
- Использовать динамический импорт

**Оценка:** 2-3 часа

#### 4. Улучшение обработки ошибок
**Проблема:**
- Нет глобального обработчика ошибок
- Некоторые функции не обрабатывают ошибки загрузки ресурсов

**Решение:**
- Добавить `window.addEventListener('error', ...)`
- Добавить try-catch в критичные функции
- Создать систему логирования ошибок

**Оценка:** 2-3 часа

### Низкий приоритет

#### 5. Минификация и сборка
**Проблема:** JavaScript и CSS не минифицированы для production.

**Решение:**
- Настроить сборку через Webpack/Rollup/Vite
- Минифицировать JS и CSS
- Использовать code splitting

**Оценка:** 4-6 часов

#### 6. SEO улучшения
**Проблема:**
- Нет структурированных данных (JSON-LD)
- Можно улучшить Open Graph теги

**Решение:**
- Добавить JSON-LD разметку для событий (Event schema)
- Улучшить Open Graph теги
- Проверить через Google Search Console

**Оценка:** 2-3 часа

#### 7. Тестирование
**Проблема:** Нет unit и E2E тестов.

**Решение:**
- Написать unit тесты для критичных функций (Jest)
- Добавить E2E тесты (Playwright/Cypress)
- Настроить CI/CD

**Оценка:** 8-12 часов

#### 8. Документация
**Проблема:** Нет JSDoc комментариев для функций.

**Решение:**
- Добавить JSDoc комментарии
- Создать документацию по архитектуре

**Оценка:** 3-4 часа
